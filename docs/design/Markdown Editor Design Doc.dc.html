Markdown Editor for macOS

A minimalist WYSIWYG markdown editor — markdown renders as you type (Typora/Bear model), with a document library sidebar, an outline panel, tabs, and rich export. Built from your four mockups. Audience: developers implementing in SwiftUI; layouts are described in terms of NavigationSplitView / adaptive columns so they port to iPadOS and iOS.

Assumptions
· App name is a placeholder ("Sample") — chrome shows document names only.
· WYSIWYG editing with syntax reveal on cursor focus (source shown for the active span).
· Multi-tab; additional windows supported, each with independent panel state.
· Content model: CommonMark + GFM tables/task lists, LaTeX math, Mermaid, footnotes, ==highlights==.
How to read this doc
· Sections 01–05 explore options — reference any badge id (e.g. 1a) in chat to pick or remix.
· Sections 06–10 are the spec: element sheet, states, flows, responsive rules, and tokens.
· All mockups are static hi-fi; annotations in orange mono are implementation notes.

Syntax reveal (WYSIWYG core rule)
**Never mutate state** from a view
When the caret enters a styled span, its markdown delimiters fade in at 35% in mono. Leaving the span hides them. Whole blocks (code, math, mermaid, tables) flip to source on double-click instead.
Core model note (SwiftUI)
Source of truth is always the markdown string + AST (swift-markdown / cmark-gfm), never attributed strings. The editor is a projection: edits mutate the AST, renderer re-projects. This is what makes tables, math, footnotes, mermaid, and lossless export possible. Documents are files-on-disk (.md) with an asset folder per library — no proprietary container. NSDocument provides autosave-in-place + version history; FSEvents watches for external edits; sync is plain iCloud Drive or any file provider (hence "synced ✓" in the sidebar footer).
Also specified (not pictured)
[TOC] block renders an inline linked list mirroring the outline panel · YAML front matter collapses to a compact metadata chip above the H1; click to edit source · smart pairs auto-close ** _ == $ ` (suspended inside code spans/fences) · external edits reload silently when the doc is clean, show a non-blocking merge banner when dirty · selection statistics: selecting text swaps the status bar to counts for the selection.

09Responsive behavior & platform ports

One layout system, three size classes. Panels collapse in priority order (outline first, then sidebar). The editor column never exceeds 680pt of text measure regardless of window width.

Format pill collapses to a single Aa button under 700pt. Tab bar switches to a compact popup menu under 560pt. Status bar drops statistics first, section name last. All thresholds are on the editor column width, not the window — so a wide window with both panels open behaves like a narrow one.

Port rules. All view models are platform-free (the AST + SessionStore core). Only navigation containers and the format-control surface differ per platform. Highlight/callout/code tokens are shared asset-catalog colors, so dark mode + iOS come for free.
