PRD Quoin · v0.1 draft 2026-07-02

Quoin

A fast, native markdown viewer for macOS and iOS.

Superseded (2026-07-02): the product scope is now a WYSIWYG markdown editor per the design handoff in docs/design/handoff.md, which is canonical for scope, UI, and interaction. This PRD remains valid for performance budgets (§05), privacy stance (§06), and the reading-feature requirements the editor inherits.

A quoin is the wedge a letterpress printer uses to lock type into the chase — the small, precise tool that makes the whole page hold. Quoin the app does the same for markdown: it takes plain text and locks it into a beautifully set, instantly rendered page, with no ceremony and no weight.

01Problem & Vision

Markdown is the default format for notes, docs, READMEs, and AI-generated output — but viewing it well on Apple platforms means choosing between Electron-based apps (slow, 200 MB+, memory-hungry), editors that bury reading under editing chrome, or bare Quick Look previews that can't handle tables, math, or diagrams.

Vision: the markdown equivalent of Preview.app — instant to open, gorgeous to read, complete in its format support, and invisible as software. Local-only, private by construction, and light enough that you never think about it running.

02Goals & Non-Goals

Goals

  • Fast above all. Cold open to rendered document in under 400 ms; re-render on file change imperceptible; 60/120 fps scrolling on any document.
  • Complete markdown. Full CommonMark + GFM: tables, task lists, footnotes, strikethrough, autolinks — plus math (LaTeX) and Mermaid diagrams.
  • Native and lean. Swift/SwiftUI throughout, zero JavaScript — math and Mermaid render through in-house native engines. No Electron, no web views, no runtime downloads.
  • One core, two platforms. A shared document engine powers the macOS app first and an iOS/iPadOS app from the same code.
  • Local-only. No accounts, no sync, no telemetry, no network access at runtime.

Non-Goals (v1)

  • Not a markdown editor in v1 — Quoin is reader-first. (The one exception: toggling task-list checkboxes writes back to the file.) The architecture is explicitly designed to grow an edit mode later, but it doesn't ship in v1.
  • No sync, cloud storage, or collaboration.
  • No plugin system or custom JS extensions.
  • No note-management/library features — Quoin opens files, it doesn't own them.

03Users & Core Use Cases

  • Developers & technical writers reading READMEs, design docs, and specs — often with tables, code, and diagrams.
  • AI-tool users reviewing generated markdown (reports, plans, transcripts) that updates on disk while they watch.
  • Note-takers who keep plain .md files in folders and want a beautiful reading surface plus quick task-checking.
  • Anyone exporting — turning a markdown file into a clean PDF or RTF to share with people who don't speak markdown.

04Feature Requirements

Priorities: P0 ships in v1.0 (macOS) · P1 fast-follow / v1.x · P2 considered, not committed.

Rendering

IDPriRequirement
R-01P0Full CommonMark + GFM rendering: headings, emphasis, lists, block quotes, code blocks, links, tables, task lists, strikethrough, footnotes, autolinks.
R-02P0Native text rendering (TextKit 2) — selectable, accessible, crisp at any size; light & dark mode with system appearance.
R-03P0Inline images: local relative/absolute paths render inline, scaled to layout width, async-loaded and cached. Remote image URLs show a placeholder by default (local-only policy) with per-document opt-in.
R-04P0Syntax highlighting for fenced code blocks (common languages, native highlighter).
R-05P0Math: inline $…$ and block $$…$$ LaTeX typeset natively (in-house TeX-subset engine) as crisp vector output; unknown commands degrade to marked literals.
R-06P0Mermaid diagrams in ```mermaid fences rendered by a native engine, phased by type: flowchart, sequence, and pie in v1.0; state/class/ER, then gantt/gitgraph/journey in updates. Unsupported types show the styled source with a notice — never an error.
R-07P1Adjustable reading theme: type size, line width, serif/sans body, custom accent.

Live behavior

IDPriRequirement
L-01P0Live rendering: the document re-renders automatically when the file changes on disk, preserving scroll position. Target <100 ms from file-save to updated view for typical documents.
L-02P0Interactive task lists: clicking a checkbox toggles it and writes the change back to the source file safely (no reformatting of untouched content).
L-03P1Follow mode: auto-scroll to the newest content when a file is being appended to (AI output, logs).

Navigation & insight

IDPriRequirement
N-01P0Table of contents sidebar built from the heading tree; click to jump; tracks scroll position; collapsible.
N-02P0Quick search (⌘F): instant as-you-type in-document search with match highlighting, count, and next/previous navigation.
N-03P0Statistics: word/character/heading counts, reading time, link & image counts, task progress (3 of 7 done) — in a summary popover, live-updating.
N-04P0Tabs: multiple documents in one window via native macOS window tabs (⌘T, drag to merge/split).
N-05P1Quick open (⌘O / ⌘P): fuzzy-find across recently opened files.
N-06P1Internal anchor links (#heading) and relative links to other .md files open in a new tab.

Export

IDPriRequirement
E-01P0PDF export: paginated, styled like the on-screen rendering, with working links and embedded images/diagrams.
E-02P0RTF export with formatting (headings, emphasis, lists, tables) for pasting into word processors.
E-03P0Plain text export: markdown syntax stripped, structure preserved.
E-04P0Markdown export: normalized/pretty-printed copy of the source.
E-05P1Print (⌘P) via the same pagination engine as PDF.

Platform

IDPriRequirement
P-01P0macOS app (macOS 14+): document-based, opens .md .markdown .txt, registered as a viewer for markdown UTIs, Quick Look-fast startup.
P-02P0Minimalist UI: content-first window, one slim toolbar, everything else behind keyboard shortcuts and menus. No panels open by default.
P-03P1iOS/iPadOS app from the same core: document browser (Files integration), full rendering parity, TOC, search, stats, share-sheet exports.
P-04P2Quick Look extension so Finder previews use Quoin's renderer.

06Privacy & Data

Everything stays on device. No network calls at runtime (remote images are opt-in per document), no analytics, no accounts. The only files Quoin writes are exports the user asks for and checkbox toggles in files the user opened.

07Release Plan

  • M1 — Reading core (macOS): parse + render CommonMark/GFM natively, tabs, TOC, search, stats, live file watching. Math/Mermaid show styled source fallback.
  • M2a — Rich content: native math typesetting, syntax highlighting, inline images, checkbox write-back.
  • M2b — Diagrams: native Mermaid engine, phase D1 (flowchart, sequence, pie). Runs parallel to M3; can slip without blocking it thanks to the fallback.
  • M3 — Output: PDF/RTF/TXT/MD export, print. → v1.0 macOS
  • M4 — iOS: document browser app on the shared core, share-sheet exports. → v1.0 iOS
  • Later: diagram phases D2/D3 (state/class/ER, gantt/gitgraph/journey); edit mode on the session architecture already in place.

08Success Criteria

  • All performance budgets met on a base-model M1 Mac and A15 iPhone.
  • 100% of CommonMark spec tests and GFM extension tests pass in the core engine.
  • A 500-page markdown book scrolls at full frame rate.
  • Zero runtime network connections observed under instrumentation.
